3.1. Design Patterns

Design Patterns (Entwurfsmuster) sind (Muster-)Lösungen für wiederkehrende Probleme beim Entwurf von Software. Sie erleichtern die Entwicklung und Architektur von großen Softwareprojekten. Dabei ist zu beachten, das es Muster sind und kein fertiger Quellcode.

Arten

Es gibt drei Arten von Design Patterns:

  • Erzeugungsmuster
  • Strukturmuster
  • Verhaltensmuster

Auflistung einiger Design Patterns

  • Accumulator
  • Class Adapter
  • Composite
  • Decorator
  • Factory
  • Iterator
  • Object Adapter
  • Observer
  • Proxy
  • Singleton
  • State
  • Strategy
  • Template Method

Eine größere Liste und weitere Informationen finden Sie auf Wikipedia.
